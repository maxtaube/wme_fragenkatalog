# Fragenkatalog ## Beschreiben sie den grundlegenden Aufbau eines HTML-Dokuments HTML-Dokument besteht aus: * Dokumententypdefinition * Wurzelelement &lt;html&gt; * head- und body-Teil Beispiel: ```html <!DOCTYPE html> <html> <head> <title>Titel</title> <meta attribut="wert" /> </head> <body> <h1>Überschrift</h1> <p>Text</p> </body> </html> ``` Innerhalb des &lt;html&gt;-Elements gibt es * das **head**-Element, in welchem für das Gesamtdokument wichtige Informationen angegeben sind (z.B. Titel, Metadaten, ...) * das **body**-Element, in dem die eigentlichen Inhalte enthalten / referenziert sind ----------------------------------------------- ## Was ist ein Doctype? Dokumententypdefinition gibt an, nach welcher Grammatik das HTML-Dokument strukturiert ist. ----------------------------------------------- ## Welche HTML-Elemente kennen Sie, um Einträge in einer Liste/Aufzählung darzustellen? Unsortierte Liste ```html <ul><!-- Unordered List --> <li>Item 1</li> <li>Item 2</li> <li>Item 3</li> </ul> ``` Sortierte Liste ```html <ol><!-- Ordered List --> <li>Item 1</li> <li>Item 2</li> <li>Item 3</li> </ol> ``` Definitionslisten ```html <dl><!-- Ordered List --> <dt>Item 1</dt><dd>Definition</dd> <dt>Item 2</dt>dd>Definition</dd> <dt>Item 3</dt>dd>Definition</dd> </dl> ``` ----------------------------------------------- ## Wie kann CSS in eine Website eingebunden werden? Extern ```html <head> <link rel="stylesheet" type="text/css" href="mystyle.css"> </head> ``` ```css body { background-color: #000000; } ``` Intern ```html <head> <style> body { background-color: linen; } h1 { color: maroon; margin-left: 40px; } </style> </head> ``` Inline ```html <h1 style="color:blue;margin-left:30px;">This is a heading.</h1> ``` ----------------------------------------------- ## Was ist ein Selektor? Der *Selektor* gibt an, für welche/e HTML-Element/e die Formatdefinition gilt und dient damit der Auswahl bestimmter HTML-Elemente ----------------------------------------------- ## Welche Arten von Selektoren kennen Sie? ```css * { Gilt für alle } h1 { gilt für alle h1 } #nav { gilt für Element mit id="nav" } li, dd, dt { gilt für alle li, dd und dt} a h1 { gilt für alle h1 innerhalb eines a} h1 + i { gilt für alle i nach einem h1} div > a { gilt für alle a direkt unterhalb eines div} p[align=left] {gilt für alle p mit align=left} .hallo {gilt für alle Elemente mit class="hallo"} usw. ``` ----------------------------------------------- ## Was ist ein attributabhängiger Selektor? Selektiert nur Elemente die ein bestimmtes Attribut besitzen oder deren Attribut einen bestimmten Wert hat. z.B.: ```css p[align=left] {gilt für alle p mit align=left} ``` ----------------------------------------------- ## Was sind Kaskaden? ### Kaskadierung bei Cascading Stylesheets (CSS) * CSS erlaubt die Nutzung mehrerer Stylesheets * Alle Stilbeschreibungen werden dann zusammegefasst auf das Dokument angewandt ----------------------------------------------- ## Was bedeutet und wie funktioniert Priorisierung in CSS? * Bei Überdeckung werden Werte nach Vorrangregeln sortiert (höchste zu niedrigste Priorität: Autor > Benutzer > Browser) * Reihenfolge: später verarbeitete Regel überschreibt frühere Regel (die Regel, die weiter unten steht wird angewandt) * Kontextregeln haben höhere Priorität als allgemeine Regeln * Selektoren mit Attributangabe haben noch höhere Prioritätodo ----------------------------------------------- ## Wie können Stile für verschiedene Ausgabemedien (Bildschirm, Druck, ...) definiert werden? Das Attribut **media** bestimmt für welches Ausgabemedium eine Datei verwendet werden soll. Die Trennung der Stile ist auch mittels mehrerer Stylesheets möglich. ```css @media print { selektor { eigenschaft: wert; } } @media screen, handheld { selektor { eigenschaft: wert; } } ``` ----------------------------------------------- ## Wie können Stilangaben vererbt werden? ```css span { color: blue; } .extra span { color: inherit; } ``` Alle **span**-Elemente erhalten die Schriftfarbe blau. **span**-Elemente innerhalb von Elementen der Klasse **extra** hingegen erben die Farbe der Klasse. ----------------------------------------------- ## Was bedeutet und wie funktioniert der Wert "inherit"? Durch **inherit** erbt ein Element den Wert der angegebenen Eigenschaft von seinem Elternelement. Das Wurzelelement kann dementsprechend keine Werte erben. Falls keine Werte geerbt werden können, werden die Standardwerte verwendet. ----------------------------------------------- ## Beschreiben Sie die Ausgabe der folgenden HTML-Seite inkl. der zugehörigen Stilangaben HTML-Seite auf der eine Überschrift gefolgt von einem kurzen Text mit Hervorhebung zu sehen ist. Die Hervorhebung ist fett gedruckt und Elemente der Klasse y sind schräg gedruckt. ----------------------------------------------- ## Was ist JavaScript? * *JavaScript* ist eine einfache, imperative Skript-Programmiersprache für meist Client-seitige Funktionalitäten, aber auch Server-seitige Funktionen (z.B. mittels node.js). * Einbettung in HTML entweder direkt (in &lt;script&gt;-Tag oder inline in Attributen) oder indirekt als externe Datei (mittels **src**-Attribut des **script**-Tag) * Typische Anwendungsgebiete: - Überprüfung von Formularen - Dynamische Interaktion (Buttons usw.) - Öffnen von Fenstern in bestimmter Größe - Verändern der Eigenschaften von Elementen * kann auf Elemente des HTML-Dokuments per DOM (Document Object Model) zugreifen * kann Dokument dynamisch ändern ----------------------------------------------- ## Was ist ein Java Applet? * wird durch Java-Klassen implementiert, die vom Server geladen werden müssen * sind externe, in HTML-Dokument eingebundene Komponenten * sind GUI-Elemente und interaktiv * Ausführung in sicherer Browser-Umgebung in Java Runtime * erlauben Kommunikation mit serverseitigen Anwendungen ----------------------------------------------- ## JavaScript-Beispiel erläutern Bei Klick auf den Knopf mit der Beschriftung "Try it" wird die Funktion **myFunction()** ausgeführt, welche welche die Browser-eigene Benachrichtigungsfunktion verwendet um den Text "Hello, how are you?" anzuzeigen. ----------------------------------------------- ## Was ist XML? XML ... eXtensible Markup Language Definierbare Auszeichnungssprache XML: * standardisierte Metagrammatik zur Definition der Syntax von Auszeichnungssprachen * ist eine Metamarkup-Sprache für kontextfreie Sprachen, ist Grammatik einer Sprache, mit der sich Regeln von Grammatiken definieren lassen ----------------------------------------------- ## Wozu wird XML verwendet? * zur Definition der Syntax von Auszeichnungssprachen ----------------------------------------------- ## Was sind besondere Eigenschaften einer XML-basierten Technologie? * maschinen- und menschenlesbar * traversierbar, da Baumstruktur / Verschachtelung * textbasiert * Trennung von Inhalt und Präsentation ----------------------------------------------- ## Welche XML-basierten Web-Technologien kennen Sie? * XHTML, HTML5 * SVG, MathML, RDF, Atom, RSS * XSLT, XPath, XML Schema, XSL-FO, XLink, XPointer ----------------------------------------------- ## Ein XML-Dokument heißt wohlgeformt, wenn es ... ? alle XML-Regeln einhält. Also: * ein Wurzelelement * immer Start- und End-Tag enthalten * ebenentreu paarige Start- und End-Tags * keine Mehrfachverwendung desselben Attributs je Tag * Attributeigenschaften in Anführungszeichen * Groß-/Kleinschreibung beachtet ----------------------------------------------- ## Ein XML-Dokument ist valide, wenn es ... ? * wohlgeformt ist * einen Verweis auf eine Grammatik enthält * und das durch die Grammatik beschriebene Format einhält ----------------------------------------------- ## Untersuchen Sie ein Dokument auf Wohlgeformtheit und Validität. ----------------------------------------------- ## Was ist ein XML-Schema? Wofür verwendet man es? * Alternative zu DTD * Definierung von Strukturen für XML-Dokumente in Form eines XML-Dokuments * Beschreibung von Datentypen ----------------------------------------------- ## Welche grundlegenden Datentypen stehen in einem XML-Schema zur Verfügung? Beispiele. *Einfache Datentypen* ohne Unterelemente oder Attribute * byte, integer, short, float, date, time, duration, language Beispiel: ```xml <xsd:simpleType name="TFahrzeuge"> <xsd:union memberTypes="TFahrrad TMotorrad TPkw TLkw ..." /> </xsd:simpleType> ``` Bildung eigener Typen durch Einschränkung, Vereinigung, Listenbildung vordefinierter Typen. *Komplexe Datentypen*, mit optionalen Attributen und Unterelementen. Beispiel: ```xml <xsd:complexType> <xsd:element>Unterelement</xsd:element> ... </xsd:complexType> <xsd:element name="Leistung" type="TLeistung" /> <xsd:complexType name="TLeistung"> <xsd:choice minOccurs="0"> <xsd:element name="PS" type="xsd:integer" /> <xsd:element name="kW" type="xsd:integer" /> </xsd:choice> </xsd:complexType> ``` ----------------------------------------------- ## Was ist eine Dokumenttypdefinition (DTD) und wofür wird sie benutzt? * Satz an Regeln der benutzt wird um Dokumente eines bestimmten Typs zu deklarieren * erfüllt gleiche Funktion wie XML Schema, aber weniger flexibel ----------------------------------------------- ## Nachteile von DTDs? * Viele Dinge nicht berücksichtigt, die für den Einsatz von XML-Dokumenten zur Datenbeschreibung und Datenaustausch wichtig sind: * Definition von impliziten Gültigkeitsregeln / Nebenbedingungen für Elemente nicht möglich * Kardinalitäten lassen sich nur umstädlich über Aufzählungen definieren * einmal definierte Inhatsmodelle können nicht dynamisch angepasst und wiederverwendet werden ----------------------------------------------- ## XPath, XLink, XPointer? ### XLink XLink definiert die Attribute, die Elementen hinzugefügt werden müssen um als Verweisquelle/-Ziel oder als Verweis zu dienen * generelles Konzept der Erstellung von Links * attributbasierte Syntax zur Definition von Links in XML-Dokumenten * definiert Syntax und Semantik der Verknüpfung über spezielle Attribute * Attribute können für jedes XML-Element definiert werden, Verwendung von Namespaces (xlink) Mehr Möglichkeiten als HTML-Links: * unabhängig von konkreter Grammatik * 1:1, bidirektionale und 1:n-Verknüpfungen * Trennung von Dokumenteninhalten und Verknüpfungen * Rollen für Verknüpfungen ### XPath XML Path Language * dient Adressierung beliebiger Knoten innerhalb desselben oder eines anderen XML-Dokuments (Elemente, Attribute, Textknoten) * Grundlage für XPointer und XSLT * operiert auf Baumstruktur * Definitionen von Achsen und Fuktionen zur Navigation * adressiert einzelne Knoten (node) und Knotenmengen (node-set) #### Achsen Durch Achsen lassen sich knoten kategorisieren. * Anchestor: alle Vorgängerknoten * Parent: erster Vorgängerknoten (Anchestor[0]) * Anchestor or self: alle Vorgänger inklusive aktueller Knoten * self: nur aktueller Knoten * child: alle direkten (ersten Grades) Nachfolgeknoten * descendant: alle Nachfolgeknoten bis zu den Blättern * following: alle Knoten rechts von der Anchestor-or-self-Achse * preceding: alle Knoten links von der Anchestor-or-self-Achse ![XPath Achsen](img/axles1.png "XPath Achsen") ![XPath Achsen](img/axles2.png "XPath Achsen") #### Syntax * "/" - Wurzelknoten * "*" - alle Elemente innerhalb eines Pfades * //Name - alle Elemente eines Namens unabhängig vom Pfad * [...] - Einschränkung der Auswahl auf z.B. erstes Element (//ccc[1]) * "|" - kombiniert Pfadbezeichner * @ - selektiert Attribute ### XPointer Kann Punkte und Bereiche adressieren (XPath nur Knoten und Knotenmengen (Elemente, Attribute, Textknoten)). ----------------------------------------------- ## Verwendung von XLink, XPoint, XPath * XLink: Verbinden von Dokumenten - Wenn beide Dokumente XML, dann in beide Richtungen möglich - Ziel muss kein XML-Dokument sein - kann in jedem Element verwendet werden - genutzt, wenn weitere Informationen zur Verknüpfung angegeben werden sollen * XPath: - zum Adressieren bestimmter Knoten oder Knotenmengen im selben oder in anderem XML-Dokument - Verwendung durch andere XML-Technologien die Elemente adressieren müssen * XPointer: - Verweisen auf Elemente innerhalb einer URI - Adressieren von Punkten oder Bereichen in einem XML-Dokument ----------------------------------------------- ## Worin unterscheiden sich XPath und XPointer? XPath kann nur Knoten bzw. Knotenmengen adressieren (Elemente, Attribute, Textknoten). XPointer dagegen kann Punkte und Bereiche adressieren. Beispiel: Definition von Linkanker als zusammenhängende, jedoch nicht wohlgeformte Textbereiche. Wichtigster Begriff: *location*. Location: * XPath-Knoten oder ein * Punkte (point) oder ein * Bereich (range) ----------------------------------------------- ## Interpretieren Sie einen XPath-Ausdruck. ----------------------------------------------- ## Was sind XSL und XSLT? XSL - eXtensible Stylesheet Language * Familie von Transformationssprachen zur Definition von Layouts von XML-Dokumenten * Unterteilung in XSL Transformations (XSLT) und XSL Formatting Objects (XSL-FO) * XSLT: zur Transformation des Quelldokuments, z.B. Extraktion, Sortierung und Nummerierung der tatsächlich anzuzeigenden Informationen * XSL-FO: dient zur Festlegung des eigentlichen Layouts des Zieldokuments ----------------------------------------------- ## Erläutern Sie die Arbeitsweise eines XSLT-Prozessors. Der XSLT-Prozessor arbeitet im Zusammenspiel mit XPath. Aus dem XML-Dokument wird ein DOM-Baum (Document Object Model) erzeugt. Auf diesem arbeiten die Prozessoren. Der XSLT-Prozessor beginnt mit dem Wurzelelement und sucht dafür Regel im Stylesheet. Die Regel wird abgearbeitet und das Ergebnis generiert. Das Quelldokument steuert damit zumindest zu Beginn die Auswahl der Regel und der damit verbundenen Transformation, sowie Werte für das Ergebnisdokument, zur Attributexpanstion oder zur Selektion von Elementen im Quelldokument. Kurz: * XML-Parser erstellt DOM-Repräsentation des Quelldokuments * XML-Parser erstellt DOM-Repräsentation des XSLT-Stylesheets * XSLTProzessor verarbeitet Anweiseungen des Stylesheets und holt dafür Nodes und Node-Sets mittels XPath-Prozessor aus dem Quelldokument (in DOM) * Erzeugung von Knoten als DOM-Repräsentation * Ausgabe als Datei in bestimmtem Format ![XSLT-Prozessor](img/xslt-processor.png "Arbeitsweise XSLT-Prozessor") ----------------------------------------------- ## Was sind Templates (XSLT)? Templates bzw. Regeln formulieren die Transformation einer Menge von Knoten des Quelldokuments. Zur Anwendung einer Transformation auf Elemente die einem bestimmten Pattern entsprechen. ----------------------------------------------- ## Was sind Pattern? (XSLT) Regel nach der eine Template Rule für einen Knoten ausgewählt wird. * im Wesentlichen eine Liste von Location Path Pattern, getrennt durch | Beispiel: ```xml imagelist/image[1] ``` passt für alle **image**-Elemente, die dem Element **imagelist** untergeordnet sind und an der ersten Position davon stehen ----------------------------------------------- ## Wofür steht XSL-FO? XSL Formatting Objects * Definition medienspezifischer Objekte für unterschiedliche Enddarstellungen * direkte Formatierung des Zieldokuments: Ausgabemedien, z.B. WWW und Druck, typisch mit Seitenbeschreibungssprachen (PCL, PDF, RTF etc.) * basiert auf DSSSL (für SGML) und CSS (für HTML) * XML-Syntax ----------------------------------------------- ## DOM? DOM ... Document Object Model * plattform- und sprachunabhängige API für die Modifikation von Dokumenteninhalten (HTML, XML) und -eigenschaften durch Programme und Skripte * definiert Standardobjekte und ein Standardmodell, das festlegt, wie diese Objekte kombiniert werden können * API definiert Art und Weise des Zugriffs und Manipulationsmöglichkeiten * ist fast vollständig in Form von Schnittstellen spezifiziert * definiert die Abbildung von XML- bzw- HTML-Dokumenten in eine Objektstruktur * definiert die Semantik dieser Schnittstellen und Objekte * repräsentiert Dokumente als Hierarchie von Knoten Unterteilt in 3 Level aus Modulen: * Level1: Core und html * Level2: Events, Views, Style, Traversal, Range * Level3: Load and Save, Validation, XPath ----------------------------------------------- ## Unterschiede zwischen DOM und XPath. * In Textknoten sieht XPath einen einzigen logischen Text, während DOM diesen unter Umständen weiter unterteilt (z.B. in verschiedene Entity-Referenzen) * während XPath einen Namespace auf jedes Element abbildet, verwaltet DOM nur die Namespace-Deklarationen * XPath unterstützt im Gegensatz zu DOM keine Bereiche ----------------------------------------------- ## Erläutern Sie die Defizite des DOM. * beim Einlesen eines XML-Dokuments werden zunächst alle darin enthaltenen Informationen im DOM-Baum repräsentiert, selbst wenn nur ein Bruchteil davon für die Anwendung relevant ist * dies kann bei umfangreichen XML-Dokumenten zu massiven Leistungs- und Speicherengpässen führen * DOM-Bäume werden sehr groß, da nicht nur Elemente sondern auch Textbestandateile ect. bei der Speicherstruktur berücksichtigt werden * die Navigation im DOM-Baum ist sehr umständlich * auch DOM Traversal und Range bringen keine wesentliche Vereinfachung ----------------------------------------------- ## Worin besteht der grundsätzliche Unterschied zwischen client- und server-seitigen Web-Technologien? Bei client-seitigen Technologien werden zum Großteil Skripte und Anwendungslogik auf dem Client, bzw. im Browser ausgeführt. Bei server-seitigen Technologien hingegen kommen Skript- und Programmiersprachen zum Einsatz die hauptsächlich auf dem Server ausgeführt werden. ### Client-seitige Web-Technologien * XHTML-Client - Klassischer Web-Server, Client kümmert sich nur um die Darstellung und die Übermittlung von UI-Ereignissen, Server bereitet HTML auf Basis der Anwendungslogik auf - Technologien: XHTML, CSS, JavaScript * Ajax - Client kümmert sich um die Präsentationslogik einschließlich Übersetzung von UI-Ereignissen auf Anwendungslogik, Interaktion vergleichbar mit Desktop-Anwendungen - Server liefert geänderte / gewünschte Teilbereiche der aktuellen UI * Plug-in - Eigenständiger Anwendungsteil, Server stellt z.B. über RPC Anwendungslogik und Daten bereit - fast ausschließlich HTTP(S)-basierte Kommunikation - zukünftig serviceorientierte Architektur und Web-Services (SOAP, WSDL, UDDI) ### Server-seitige Web-Technologien häufig verwendetes Prinzip: * Aufruf einer Server-seitigen Anwendung über standardisierte API * Realisierung der serverseitigen Anwendung über Skriptsprachen oder webspezifische Programmiersysteme serverseitige HTML-Generierung: * offline durch Vorberechnung von HTML-Seiten - Generierung statischer Seiten * Aufruf einer HTML-generierenden Anwendung - CGI - APIs, z.B. Java Servlets * Einbettung HTML-generierender Skriptteile - Interpretiert: + Server Side Scripting + Active Server Pages + PHP - Kompiliert: + Java Server Pages * XML-Weg ----------------------------------------------- ## Was ist eine Web-Architektur? * sie beschreibt die Struktur, d.h. die Komponenten, Schnittstellen und Beziehungen von Softwaresystemen einer verteilten Web-Anwendung * betrachtet sowohl statische als auch dynamische Aspekte und kann daher als Bauplan und Ablaufplan eines Softwaresystems gesehen werden ----------------------------------------------- ## Erläutern Sie den Begriff Schichten-Architektur. Dabei werden Web-Anwendungen in mehrere Schichten gegliedert, das Informationssystem also in mehrere Funktionsblöcke aufgeteilt. Ermöglicht Bildung eines Anwendungssystems aus seperaten Komponenten, welche durch eine Middleware koordiniert werden müssen. Einfachste Form: Server + Client (2 Schichten) ----------------------------------------------- ## Was bedeutet 2-Tier-Architektur? de.: 2-Schichten-Architektur * Aufteilung eines Informationssystems in die Funktionsblöcke: Präsentation, Anwendungslogik und Ressourcenverwaltung * bei 1-Tier-Architektur ist Präsentation Teil des Servers * bei 2-Tier-Architektur ist Präsentation Teil des Clients * die Anwendungslogik kann aus einer Vielzahl Services gebildet werden * über einheitliche Service-API werden diese dann dem Presentation Layer zur Verfügung gestellt Bsp.: Client-Server-Architektur * Zugriff auf: - statische, dynamische HTML-Seiten, - Datenbank-Informationen, - Dienste, z.B.: Datenbankverschlüsselung * Geeignet für einfache Web-Anwendungen * Anwendung mit vielen Client-Zugriffen, komplexen Anwendungsabläufen, Zugriff auf Legacy-Systeme, Medienströmen usw. erfordert aus Gründen der Performance eine mehrschichtige Architektur ----------------------------------------------- ## Was ist HTTP und wofür wird es eingesetzt? HTTP ... Hypertext Transfer Protocol Ist ein zustandloses Protokoll zur Übertragung von Web-Ressourcen wie z.B. HTML-, CSS- und JavaScript-Dokumenten. ----------------------------------------------- ## Beschreiben Sie wesentliche HTTP-Eigenschaften. * verwendet fast ausschließlich TCP (Transmission Control Protocol), aber auch UDP (User Datagram Protocol), welche auf dem IP-Protokoll aufsetzen * HTTP-Transaktion verläuft in 4 Schritten: - Client baut TCP-Verbindung auf - Client sendet HTTP-Request zu Server - Server sendet HTTP-Response zum Client - Server baut Verbindung zum Client wieder ab * **zustandslos**, d.h. Server betrachtet Anfrage unabhängig von vorhergehenden Anfragen * Bidirektionale Übertragung ist möglich * Browser und Server können Merkmale für folgenden Datentransfer aushandeln ----------------------------------------------- ## Was ist eine Ressource im WWW? Erläutern Sie den Begriff HTTP-Adressierung! Eine Ressource kann alles sein was weltweitig eindeutig über eine URI identifiziert werden kann. Diese URI dient der HTTP-Adressierung um die Ressource ausfindig zu machen. Sie ist identifizierbar mit Speicherort, Name und Darstellungsart (integrierte Anwendung). ----------------------------------------------- ## Erläutern Sie die Adressierung einer Ressource im Web. Eine Ressource wird adressiert indem Sie eine URI zugewiesen bekommt. Diese dient als Information zur Identifizierung und letztlich Lokalisierung im WWW. ----------------------------------------------- ## Was ist eine URI? Was ist eine URL? Was bedeuten die Begriffe? Erläutern Sie die Unterschiede zwischen URI und URL! URI ... Uniform Ressource Identifier Eine URI ist eine weltweit eindeutige Zeichenkette zur Identifizierung einer Web-Ressource. Sie besteht entweder aus einer URL oder einer URN. URL ... Uniform Ressource Locator URL-Format: ```http protocol://[user:[password]@]hostname:[port]/documentname[?parameter] ``` URL besteht meist aus: * Zugangsprotokoll:// Hostadresse:Port/Pfad * Zugangsprotokoll (Präfix): http://, ftp://, mailto://, file:// usw. * Hostadresse: IP-Adresse oder DNS-Name (Domain Name Service) (jede DNS wird auf eine IP abgebildet) * Port: Optional, *:&lt;Portnummer&gt;*, Standard: Port 80 * Pfad zur Ressource: leerer Pfad ist Wurzel **/** Beispiel: ```html http://www.limbusdev.org:80 ``` URL ist eine Unterart der URI. URN ... Uniform Ressource Name ----------------------------------------------- ## Beschreiben Sie das HTTP-Nachrichtenformat! Client-Request: * Methode, URL, HTTP-Version * (Optional) Allgemeiner Header * (Optional) Request-Header * (Optional) Body-Header * (Optional) Body Server-Response: * HTTP-Version, Statuscode, Information * (Optional) Allgemeiner Header * Response-Header * (Optional) Body-Header * (Optional) Body Body ist durch Leerzeile vom Rest getrennt. Beispiel Request-Header: ```http Connect to 87.106.6.173 on port 80 ... ok GET / HTTP/1.1[CRLF] Host: www.limbusdev.org[CRLF] Connection: close[CRLF] User-Agent: Web-sniffer/1.1.0 (+http://web-sniffer.net/)[CRLF] Accept-Encoding: gzip[CRLF] Accept-Charset: ISO-8859-1,UTF-8;q=0.7,*;q=0.7[CRLF] Cache-Control: no-cache[CRLF] Accept-Language: de,en;q=0.7,en-us;q=0.3[CRLF] Referer: http://web-sniffer.net/[CRLF] ``` ----------------------------------------------- ## Was ist ein HTTP-Header? Was ist ein Request-Header? Der HTTP-Header enthält Daten und Parameter des Request- bzw. Response-Header. Der allgemeine Header übermittelt Informationen zur Nachricht selbst: * Connection: Anforderung an den Server, die TCP-Verbindung offen zu lassen * Date: Zeitpunkt, an dem die Message erzeugt wurde Request-Header kommt vom Client, enthält Informationen zum Request, Client oder User: * Authorization: Identität des Benutzers * Referer: URL des HTML-Dokuments, aus dem der Request generiert wurde * User-Agent: Kennung des Browsers > Anpassung der Dokumente an diesen * If-Modified-Since: enthält Zeitstempel für bedingte Anfrage einer Ressource, z.B. wenn diese bereits beim Client oder Proxy im Cache liegt Response-Header enthält Informationen die der Server an den Client übermittelt: * Location: URL, die Browser anstelle der ursprünglich erfragten anfordern soll * Server: Kennung des Servers, d.h. serverseitig verwendete Software * Expires: Verfallsdatum der Ressource * Cache-Control: Beeinflussung des Caching-Verhaltens * WW-Authenticate: enthält bei einem Response mit **Status 401** **Unauthorized** die Spezifikation des gewünschten Authentifizierungsverfahrens ----------------------------------------------- ## Nennen und beschreiben Sie HTTP-Methoden. * **OPTIONS** liefert Informationen über Kommunikationsoptionen einer über URL adressierten Ressource bzw. des Servers * **GET** fordert eine Ressource (Web-Site, Datei) an. Parameter sind die URL der angeforderten Ressource, HTTP-Version und an die URL angehängte Daten - Benutzereingaben als *name=wert*-Paare, getrennt durch *&amp;* als Zeichenkette - Daten MIME-Type-kodiert, z.B. aus Leerzeichen wird +, Sonderzeichen hexadezimal * **HEAD** äquivalent zu **GET**, jedoch nur Rückkgabe des HTTP-Response-Header der Ressource, keine Daten * **POST** überträgt Parameter zu einer durch die URL angegebenen Ressource auf dem Server - URL beschreibt meist ein Programm auf dem Server, das Daten weiter verarbeiten soll (Formulardaten u.Ä.) * **PUT** überträgt Daten, meist vollständige HTML-Dokumente, zum Server - Daten im Gegensatz zu **POST** unter bestimmter URL abgespeichert * **DELETE** löscht die in der URL übergebene Ressource vom Server * **TRACE** liefert gesendete Nachticht sowie den Übertragungsweg zurück ----------------------------------------------- ## Was bedeutet Caching? RIA ... Rich Client Anwendung Computer in einer verteilten Umgebung, der verteilte Ressource, aber auch lokale Ressourcen nutzt. Ziel sind desktop-ähnliche Interaktionsmöglichkeiten auch bei Web-Anwendungen. RIA erkennt man daran, dass * sie Benutzeroberflächen anbieten, die reich an Interaktionsmöglichkeiten sind, * sie entweder über das Internet kommunizieren (zum Beispiel mit Servern) oder zumindest über dieses ausgeliefert werden. ----------------------------------------------- ## Welche Methoden gehören zu einem [XMLHttpRequest](https://wiki.selfhtml.org/wiki/JavaScript/XMLHttpRequest "XMLHttpRequest")-Objekt? * abort(): aktuell laufenden Request abbrechen * getAllResponseHeader(): Liste aller Header als Key-Value-Paare * getResponseHeader(name): Wert für angegebenen header * open(method.url [syncFlag, username, password]): Verbindung zur Zieladresse aufbauen, Art der Übertragung vereinbarem * send(body | null): sendet Request an Server Beispiel: ```javascript xhttp = new XMLHttpRequest(); xhttp.open("GET", "ajax_info.txt", true); xhttp.send(); ``` ----------------------------------------------- ## Portierung 10 Jahre alter dynamischer Web-Anwendung (XHTML, CSS, JavaScript, JavaServlets) auf Ajax. Wo eingreifen? * XHTML auf HTML5 portieren (fast komplett kompatibel) * JavaScript überarbeiten um asynchron statt synchron das DOM zu manipulieren * entsprechende Antwortfunktionen in den JavaServlets anlegen ----------------------------------------------- ## Moderne Frameworks für Web 2.0 / Ajax-Anwendungen? * Ruby on Rails * Node.js * OpenLaszlo * Eclipse RAP * Google GWT * Qooxdoo * ExtJS * Wicket Auswahl nach: * Szenario und Anforderungen * Entwicklungszeit, -Aufwand, Erfahrung * Client-Server-Verteilung * Performanz und Skalierbarkeit * Aufwand bei Deployment und Wartung * Verbreitung, Reife, Entwicklung, Standardkonformität ----------------------------------------------- ## Wie vereinfachen Web-Frameworks den Entwicklungsprozess von Web-Anwendungen? * Wiederverwendung von Code * schnelle Entwicklung lauffähiger Anwendungen * Komplexitätsreduktion * Templates * Umsetzung von populären Design Patterns (MVC) ----------------------------------------------- ## Aspekte Client-Server-Aufteilung bei RAP und GWT? Ausführung im Browser und damit kein unmittelbarer Zugriff auf die OS-Ebene. Serverteil in Java, Clientteil in JavaScript. Nach Start je eine Instanz von jedem GUI-Widget auf Client und Server. Synchronisierung von Client und Server muss gesichert sein. Nur senden sich ändernder Daten. ----------------------------------------------- ## Unterschiede Eclipse RAP und Google GWT? ### [RAP](https://de.wikipedia.org/wiki/Remote_Application_Platform "RAP") (Remote Application Platform) * Java-Code komplett auf Server ausgeführt * statische JavaScript-Bibliothek vom Client geladen * ständige Synchronisation zwischen Client und Server (vollautomatisch) * serverseitige Auswertung von Interaktionen und Ausführung von Logik * stärkere Netzwerklast zwischen Client und Server ### [GWT](http://www.gwtproject.org/overview.html "GWT") (Google Web Toolkit) * Java-Code wird in JavaScript umgewandelt * Anwendungszustand liegt auf dem Client * Kommunikation mit Server nur für Datenzugriff (Tausch und Skalierung von Server möglich) * clientseitige Ausführung der Logik * aufwendigere Erstellung, aber flexibler * leistungsstärker durch Caching und geringere Kommunikation # Quellen Hauptsächlich aus dem WME Skript von Professor Raimund Dachselt.
